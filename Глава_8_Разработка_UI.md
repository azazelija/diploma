# 8. РАЗРАБОТКА ПОЛЬЗОВАТЕЛЬСКОГО ИНТЕРФЕЙСА

В рамках данного этапа было реализовано несколько ключевых компонентов пользовательского интерфейса, обеспечивающих удобное взаимодействие с системой управления задачами. Интерфейс разработан с применением современных веб-технологий и ориентирован на интуитивность использования.

Для построения пользовательского интерфейса выбран фреймворк Next.js версии 14.1.0 совместно с библиотекой React 18.2.0. Данное решение позволяет создавать динамичные интерфейсы с поддержкой серверного рендеринга и оптимизированной производительностью. Все компоненты написаны на языке TypeScript, что обеспечивает строгую типизацию и снижает количество ошибок на этапе разработки.

Основой визуального представления выступает главная страница приложения, размещённая в файле page.tsx. На данной странице организовано управление состоянием приложения с использованием хуков React, включая состояния задач, статусов, текущего пользователя и модальных окон. При монтировании компонента выполняется проверка авторизации пользователя и загрузка необходимых данных с сервера.

Центральным элементом интерфейса является компонент KanbanBoard, реализующий канбан-доску для визуализации задач. Доска разделена на колонки в соответствии со статусами задач. Каждая колонка отображает название статуса с цветовым индикатором и количество задач в данном статусе. Реализована функция фильтрации задач по статусам, что позволяет группировать их в соответствующих колонках. Для каждого статуса предусмотрена русскоязычная локализация названий: «Нужно сделать», «В процессе», «На проверке», «Готово», «Отменено». В случае отсутствия задач в колонке выводится соответствующее сообщение.

Отображение отдельных задач осуществляется через компонент TaskCard. Карточка задачи содержит заголовок, описание, индикатор приоритета и срок выполнения. Для приоритетов используется визуальное обозначение эмодзи-иконками: низкий приоритет отмечен синей меткой, средний — оранжевым ромбом, высокий — красным кругом, а срочный — огнём. Срок выполнения форматируется с помощью библиотеки date-fns с поддержкой русской локализации. Реализована проверка просроченных задач: если срок истёк, дата выделяется специальным стилем. В правом верхнем углу карточки расположены кнопки редактирования и удаления задачи.

Навигационная панель представлена компонентом Header, размещённым в верхней части интерфейса. Панель включает логотип приложения «Task Manager» с эмодзи-иконкой, строку поиска задач, а также область действий пользователя. В зависимости от состояния авторизации отображаются различные элементы управления. Для авторизованного пользователя доступна кнопка создания новой задачи и выпадающее меню профиля, в котором отображается имя пользователя, адрес электронной почты и кнопка выхода из системы. Для неавторизованных пользователей предусмотрена кнопка входа в систему.

Взаимодействие с формами данных организовано через модальные окна. Компонент CreateTaskModal обеспечивает создание новых задач. При открытии модального окна пользователю предоставляется форма с полями для ввода названия, описания, выбора статуса, приоритета и установки срока выполнения. Модальное окно открывается поверх основного содержимого с затемнённым фоном, что фокусирует внимание пользователя на выполняемом действии.

Для процессов авторизации и регистрации разработан компонент AuthModal, поддерживающий переключение между режимами входа и регистрации. В режиме входа пользователю предлагается ввести адрес электронной почты и пароль. В режиме регистрации дополнительно запрашивается имя пользователя и подтверждение пароля. Переключение между режимами осуществляется без закрытия модального окна, что повышает удобство использования. После успешной авторизации или регистрации модальное окно закрывается автоматически, и происходит обновление данных пользователя.

Все формы оснащены валидацией данных на клиентской стороне. Проверяется заполненность обязательных полей, корректность формата электронной почты, совпадение паролей при регистрации. В случае возникновения ошибок пользователю выводятся информативные сообщения, помогающие устранить проблему.

Стилизация компонентов выполнена с применением CSS-модулей, что обеспечивает изоляцию стилей и предотвращает конфликты имён классов. Для каждого компонента создан отдельный файл стилей с расширением module.css. Использован современный подход к вёрстке с применением Flexbox и Grid для адаптивного расположения элементов. Цветовая схема выдержана в сдержанных тонах с акцентами на важных элементах управления.

Интерактивность интерфейса достигается за счёт обработки событий пользователя. Реализованы следующие действия:
1.	Создание новой задачи.
2.	Редактирование существующей задачи.
3.	Удаление задачи.
4.	Изменение статуса задачи.
5.	Вход в систему.
6.	Регистрация нового пользователя.
7.	Выход из системы.

Все операции выполняются асинхронно с отправкой запросов на серверную часть приложения через API-маршруты Next.js. При успешном выполнении операции данные на странице обновляются без полной перезагрузки, что обеспечивается механизмом состояния React. В случае возникновения ошибок при взаимодействии с сервером пользователю выводится соответствующее уведомление.

Для повышения удобства восприятия информации применена локализация текстовых элементов на русский язык. Локализованы названия статусов, приоритетов, сообщения об ошибках и подсказки. Форматирование дат выполнено с учётом российских стандартов представления времени.

Архитектура компонентов построена по принципу повторного использования. Общие элементы, такие как формы и модальные окна, выделены в отдельные компоненты, что упрощает поддержку кода и позволяет быстро вносить изменения. Каждый компонент получает данные через свойства (props) и взаимодействует с родительским компонентом через функции обратного вызова (callbacks). Это обеспечивает однонаправленный поток данных и предсказуемое поведение приложения.

Оптимизация производительности достигается за счёт использования клиентских компонентов (client components) только там, где необходима интерактивность. Директива 'use client' применена для компонентов, использующих хуки состояния и эффектов. Это позволяет серверным компонентам Next.js выполнять рендеринг на стороне сервера, что ускоряет начальную загрузку страницы.

В результате проделанной работы создан функциональный и отзывчивый пользовательский интерфейс, соответствующий современным стандартам веб-разработки. Интерфейс обеспечивает полный цикл управления задачами с поддержкой авторизации пользователей.

# 2.3 ВЫБОР ТЕХНОЛОГИЙ И ОБОСНОВАНИЕ СРЕДСТВ РАЗРАБОТКИ

При выборе технологий учитывались критерии производительности, надёжности, простоты поддержки и соответствия современным стандартам веб-разработки. Основным требованием являлась единая технологическая база для клиентской и серверной частей приложения.

В качестве основного фреймворка выбран Next.js версии 14. Ключевые преимущества: единая кодовая база для клиента и сервера, встроенная система маршрутизации на основе файловой структуры, серверные компоненты (Server Components) для ускорения начальной загрузки, встроенная поддержка API-маршрутов без необходимости отдельного backend-сервера. Фреймворк автоматически оптимизирует производительность через разделение кода, ленивую загрузку компонентов и предварительную загрузку ресурсов. Для системы управления задачами используется серверный рендеринг, обеспечивающий актуальность данных при каждом запросе.

Язык программирования TypeScript выбран для всех частей приложения. TypeScript добавляет статическую типизацию к JavaScript, что позволяет выявлять ошибки на этапе разработки. Система типов обеспечивает автодополнение в редакторе и предотвращает типовые ошибки при передаче данных между компонентами. Для объектов задач, пользователей и статусов определены интерфейсы с явным указанием типов всех полей.

Для управления состоянием используется библиотека React версии 18. React предоставляет компонентную модель разработки, где хуки (useState, useEffect, useCallback) обеспечивают управление состоянием. Виртуальный DOM эффективно обновляет только изменившиеся части интерфейса. Состояние системы включает список задач, статусы, информацию о пользователе, состояние модальных окон и параметры фильтрации.

Для работы с базой данных PostgreSQL используется библиотека pg (node-postgres). Это официальный клиент для Node.js с поддержкой пула соединений и параметризованных запросов, защищающих от SQL-инъекций. Модуль lib/db.ts инкапсулирует логику подключения и предоставляет единую функцию query для выполнения запросов.

Для хеширования паролей применяется библиотека bcrypt с коэффициентом сложности 10. Алгоритм включает соль и устойчив к атакам перебором. Проверка пароля осуществляется функцией compare без возможности обратного преобразования хеша.

В качестве среды разработки выбран Visual Studio Code с встроенной поддержкой TypeScript, автодополнением, проверкой типов и отладчиком. Для контроля версий используется Git. Управление зависимостями осуществляется через npm, файл package.json содержит список библиотек и скрипты для сборки и запуска приложения.

Выбранный стек технологий обеспечивает современную, производительную и поддерживаемую разработку. TypeScript создаёт единую технологическую базу, Next.js предоставляет готовые решения для веб-разработки, PostgreSQL и pg обеспечивают надёжное хранение данных. Инструменты разработки являются индустриальными стандартами с обширной документацией.

# 10. ТЕСТИРОВАНИЕ СИСТЕМЫ

Для проверки корректности работы разработанного приложения было проведено функциональное тестирование всех основных возможностей системы управления задачами. Тестирование выполнялось в веб-браузере с использованием реального окружения, включающего базу данных PostgreSQL и серверную часть Next.js.

## Авторизация и регистрация пользователей

Первым этапом проверки стало тестирование механизма авторизации. На рисунке 1 представлена форма входа в систему, где пользователю необходимо ввести адрес электронной почты и пароль. Форма содержит валидацию полей и выводит сообщения об ошибках при некорректном вводе данных. Реализовано переключение между режимами входа и регистрации без перезагрузки страницы.

[Место для вставки рисунка 1]

На рисунке 2 показана форма регистрации нового пользователя. Пользователь вводит имя, адрес электронной почты, пароль и подтверждение пароля. Проверяется совпадение введённых паролей и корректность формата электронной почты. После успешной регистрации происходит автоматический вход в систему и перенаправление на главную страницу с канбан-доской.

[Место для вставки рисунка 2]

## Просмотр и организация задач

На рисунке 3 представлена главная страница приложения с канбан-доской после успешной авторизации. Доска разделена на колонки, соответствующие статусам задач: «Нужно сделать», «В процессе», «На проверке», «Готово», «Отменено». Каждая колонка отображает цветовой индикатор статуса и количество задач. В верхней части страницы расположена навигационная панель с логотипом приложения, строкой поиска, кнопкой создания задачи и меню пользователя.

[Место для вставки рисунка 3]

Карточки задач содержат всю необходимую информацию: название, описание, приоритет с визуальным обозначением, срок выполнения и имя создателя. На рисунке 4 показаны карточки задач с различными приоритетами. Низкий приоритет отмечен синей меткой, средний — оранжевым ромбом, высокий — красным кругом, срочный — символом огня. Просроченные задачи выделяются особым цветом даты выполнения.

[Место для вставки рисунка 4]

## Создание новой задачи

Процесс создания задачи представлен на рисунке 5. При нажатии кнопки «Создать» в навигационной панели открывается модальное окно с формой создания задачи. Форма включает поля для ввода названия задачи (обязательное поле), описания, выбора приоритета из выпадающего списка, выбора начального статуса и установки срока выполнения через календарь. Модальное окно отображается поверх основного содержимого с затемнённым фоном.

[Место для вставки рисунка 5]

После заполнения формы и нажатия кнопки «Создать задачу» происходит отправка данных на сервер. В случае успешного создания задачи модальное окно закрывается, и новая задача немедленно появляется в соответствующей колонке канбан-доски. На рисунке 6 показана канбан-доска с только что созданной задачей, отображённой в колонке «Нужно сделать».

[Место для вставки рисунка 6]

## Редактирование задачи

Функциональность редактирования задачи демонстрируется на рисунке 7. При нажатии кнопки редактирования (иконка карандаша) в правом верхнем углу карточки задачи открывается модальное окно с предзаполненной формой. Все поля содержат текущие значения задачи и доступны для изменения. Пользователь может обновить любое поле: название, описание, приоритет, статус или срок выполнения.

[Место для вставки рисунка 7]

На рисунке 8 представлен результат сохранения изменений. После нажатия кнопки «Обновить задачу» модифицированные данные отправляются на сервер, и карточка задачи обновляется с новыми значениями. Если был изменён статус задачи, карточка автоматически перемещается в соответствующую колонку канбан-доски.

[Место для вставки рисунка 8]

## Изменение статуса задачи

Быстрое изменение статуса задачи показано на рисунке 9. Данная функция позволяет оперативно переводить задачи между этапами выполнения без открытия формы редактирования. При изменении статуса карточка плавно перемещается из одной колонки в другую, количество задач в колонках автоматически обновляется.

[Место для вставки рисунка 9]

## Удаление задачи

На рисунке 10 продемонстрирован процесс удаления задачи. При нажатии кнопки удаления (иконка корзины) в правом верхнем углу карточки появляется диалоговое окно подтверждения действия с вопросом «Вы уверены, что хотите удалить эту задачу?». Данная мера предосторожности предотвращает случайное удаление важных задач.

[Место для вставки рисунка 10]

После подтверждения удаления задача исчезает из канбан-доски, соответствующая запись удаляется из базы данных. На рисунке 11 показано состояние канбан-доски после удаления задачи. Количество задач в колонке автоматически уменьшается, визуальное представление обновляется без перезагрузки страницы.

[Место для вставки рисунка 11]

## Меню пользователя и выход из системы

Функциональность работы с профилем пользователя представлена на рисунке 12. При нажатии на иконку пользователя в правом верхнем углу навигационной панели открывается выпадающее меню. Меню отображает имя пользователя и адрес электронной почты, а также содержит кнопку выхода из системы.

[Место для вставки рисунка 12]

На рисунке 13 показано состояние приложения после выхода пользователя из системы. Канбан-доска и кнопка создания задачи становятся недоступными, отображается кнопка «Войти» для повторной авторизации. Это обеспечивает защиту данных и предотвращает несанкционированный доступ к задачам.

[Место для вставки рисунка 13]

## Обработка ошибок

Тестирование включало проверку обработки ошибочных ситуаций. На рисунке 14 представлены сообщения об ошибках при некорректном вводе данных в формах. Валидация выполняется как на клиентской, так и на серверной стороне. При попытке создать задачу без названия выводится сообщение «Title is required». При некорректном формате электронной почты пользователь получает соответствующее предупреждение.

[Место для вставки рисунка 14]

На рисунке 15 показана обработка сетевых ошибок. При отсутствии соединения с сервером или сбое в базе данных пользователю выводится информативное сообщение об ошибке: «Ошибка загрузки данных. Проверьте подключение к базе данных». Это позволяет пользователю понять причину проблемы и предпринять соответствующие действия.

[Место для вставки рисунка 15]

## Адаптивность интерфейса

Тестирование показало корректную работу приложения в различных условиях. Интерфейс остаётся функциональным и удобным при работе с большим количеством задач. Колонки канбан-доски поддерживают прокрутку при наличии множества карточек. Поиск по задачам работает оперативно без задержек.

Все операции создания, чтения, обновления и удаления задач выполняются корректно. Данные синхронизируются между клиентской и серверной частью без расхождений. Временные метки создания и обновления задач фиксируются точно. Локализация интерфейса на русский язык применена последовательно во всех элементах.

В результате тестирования подтверждена работоспособность всех функциональных возможностей системы управления задачами. Приложение обеспечивает стабильную работу, корректную обработку пользовательских действий и адекватную реакцию на ошибочные ситуации.
